# Contributing to pano-viewer

Thank you for your interest in contributing! Here is everything you need to get started.

---

## Development Setup

```bash
git clone https://github.com/yourusername/pano-viewer.git
cd pano-viewer
npm install
npm start   # starts the dev server at http://localhost:9000
```

---

## Project Structure

```
pano-viewer/
├── src/                # Library source (TypeScript)
│   └── index.ts        # Public API entry point
├── demo/               # Local demo app
│   └── public/         # Static assets, GIF snapshots
├── dist/               # Built output (git-ignored, generated by npm run build)
├── docs/               # Generated API docs (npm run docs)
├── esbuild.config.ts   # Build configuration
└── package.json
```

---

## Submitting Changes

1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feat/your-feature
   ```

2. **Write clear, focused commits** using [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat: add support for WebXR`
   - `fix: resolve camera drift on retina displays`
   - `docs: improve API table in README`

3. **Lint before pushing**:
   ```bash
   npm run lint
   ```

4. **Open a Pull Request** against `main` with a clear description of what changes and why.

---

## Reporting Bugs

Please open an [issue](https://github.com/yourusername/pano-viewer/issues) and include:

- pano-viewer version
- Browser + OS
- A minimal reproduction (CodeSandbox link preferred)
- Screenshot or screen recording if visual

---

## Requesting Features

Open an issue with the label `enhancement`. Describe:

- The use-case you are trying to solve
- Any API shape you have in mind
- Whether you would be willing to implement it

---

## Code Style

- TypeScript strict mode is enabled
- Prettier + ESLint rules are enforced via `npm run lint`
- Avoid adding `console.log` calls in library code; use the `loglevel` logger already in the project
